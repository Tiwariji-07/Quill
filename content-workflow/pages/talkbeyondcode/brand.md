# Talk Beyond Code brand guide

Version: 1.0  
Updated: 2026-06-10

## Idea

Talk Beyond Code is four engineers talking about everything code cannot say: opinions, experiences, practices, failures, and lessons from real software work.

The identity fuses two symbols:

- Code brackets `< >` for the craft
- A voice waveform for the conversation

The waveform sits between the brackets. The talk happens beyond the code.

`Ctrl+Shift+AI` is the podcast sub-brand. It extends the system with a keycap motif: two familiar keys plus one new key, AI as the new keystroke every engineer is learning.

## Public naming

- Public name: `Talk Beyond Code`
- Handle: `talkbeyondcode`
- Podcast: `Ctrl+Shift+AI`
- Avoid using `talkbeyondcode` as the display name on public pages.

## Logo system

Core mark: the voice between brackets.

- Brackets carry the brand gradient.
- Waveform bars are white on dark, ink on light.
- Use the tiled rounded-square mark for avatars and app icons.
- Use the bare mark inside banners, lockups, thumbnails, and overlays.

Wordmark:

- `Talk` and `Code` are white on dark or ink on light.
- `Beyond` uses the brand gradient inside the logo artwork.
- Do not use gradient text for ordinary post headings or body copy.

## Color

- Void: `#07090F`, backgrounds
- Surface: `#0D1322`, cards and keycaps
- Signal Cyan: `#22D3EE`, links and accents
- Neural Violet: `#8B5CF6`, core brand
- Spark Magenta: `#E879F9`, highlights
- Signal White: `#F8FAFC`, primary text
- Muted: `#8B96AD`, secondary text
- Brand Gradient: cyan to violet to magenta

Rule of thumb:

- 90 percent dark
- 10 percent gradient
- The gradient is the signature. Use it for one hero element per layout: brackets, the AI key, or the word Beyond inside the logo.
- Do not use large gradient backgrounds.

## Typography

- Display: Space Grotesk, headlines, wordmark, episode titles, weight 700
- Technical: JetBrains Mono, keycaps, labels, all-caps micro-copy
- Body: Inter, descriptions, captions, long-form text, weights 400 to 600

If the brand starts feeling too generic, test Mona Sans or Hubot Sans for display/body while keeping JetBrains Mono for keycaps.

## Voice

- Opinionated, not preachy.
- Practitioner-first.
- Real stacks, real failures, real prompts.
- No hype-cycle language.
- Say what worked for us, not the right way.
- Say lessons, not learnings, unless quoting someone directly.
- Lowercase energy, uppercase labels.
- Labels can be spaced caps. Body copy should stay normal sentence case.

Sample tagline:

```text
opinions · experiences · practices · beyond the code
```

Podcast one-liner:

```text
how real engineers ship with AI
```

## Banned or weak phrases

- revolutionary
- game-changing
- unlock
- 10x
- future-proof
- thought leadership
- best practices say
- welcome to our journey
- support us by liking and sharing

## Do

- Keep backgrounds near-black with one subtle glow.
- Let the AI keycap be the loudest element only when promoting `Ctrl+Shift+AI`.
- Use the bare mark at small sizes.
- Keep clear space around the mark at least the height of one waveform bar.
- Use host, guest, and team photos on LinkedIn. People build trust faster than logos.

## Do not

- Place the gradient on body text.
- Stretch, rotate, or outline the mark.
- Use pure black. Use Void `#07090F`.
- Add extra colors. The palette is closed.
- Use static posters as the only launch content.
- Make every platform post look identical.
