# Talk Beyond Code LinkedIn page

URL: https://www.linkedin.com/company/talkbeyondcode/  
Platform: LinkedIn company page  
Primary goal: build credibility and distribution for Talk Beyond Code and the `Ctrl+Shift+AI` podcast.

## Profile setup

- Display name: `Talk Beyond Code`
- Handle/slug: `talkbeyondcode`
- Logo: `400 x 400`
- Company page cover: `4200 x 700`
- Cover line: `Engineering conversations beyond syntax.`
- Cover subline: `AI · Systems · Careers · Product Thinking`
- Button: YouTube channel or latest episode
- Website: https://www.youtube.com/@talkbeyondcode
- Location: India, or the actual operating city if the team wants a local signal

## Positioning

Talk Beyond Code is a creator-led engineering page by four engineers. It should feel like real engineers comparing notes, not a media brand pretending to be bigger than it is.

The page should make three things clear quickly:

1. The brand is run by engineers.
2. The topics go beyond syntax.
3. The podcast is the flagship format, but LinkedIn posts should stand alone.

## Page pillars

Every LinkedIn page post should fit exactly one pillar.

| Pillar | Purpose | Typical formats |
|---|---|---|
| `launch` | Announcements, milestone posts, episode releases | Image post, native video, short text |
| `behind-the-scenes` | Human proof, production process, team moments | Photo, short reflection |
| `episode-lesson` | One idea from an episode that stands alone | Text post, carousel, native clip |
| `engineering-opinion` | Point of view on AI, systems, work, careers | Text post, poll |
| `audience-building` | Questions, polls, topic selection, lightweight prompts | Poll, short text |

## Content rules

- The company page is the official source. Personal profiles are the distribution layer.
- Do not copy-paste the same post across all four personal profiles.
- Native LinkedIn assets first: photo, document, or video.
- Use YouTube links sparingly. The official launch post can include the episode link. Other posts should usually place the link in the first comment.
- Do not ask for likes, shares, or support.
- Do not manufacture quotes if there is no transcript.
- Prefer specific angles over broad podcast promotion.
- Reply from the page during the first 60 to 90 minutes after posting.
- Track questions from comments as future post ideas.

## Asset choices

Easy assets that work for launch:

- Production team selfie
- Launch poster
- Guest intro card
- Team intro card
- Poll
- Simple PDF carousel
- Podcast thumbnail
- Native teaser clips when available

Use one static poster per launch moment. Repeated posters make the page look like an announcement board.

## Operating cadence

Launch weeks can be heavier. Normal weeks should stay sustainable.

- 2 to 3 company page posts per week
- 1 native video or clip when available
- 1 discussion post or poll
- 1 standalone lesson, carousel, or opinion post
- Daily comment/reply blocks during active launches

## Metrics to track

For each published post, update `index.json` after the numbers settle.

- Impressions or reach
- Likes
- Comments
- Reposts
- Link clicks when available
- Follows attributed to the post when visible
- Qualitative signal: best comment, topic request, disagreement, or useful reply

## Launch-day operating rule

For launch moments, do not post and disappear.

The first 90 minutes are for:

- Page replies
- Host comment
- Guest comment
- Team comments
- Follow-up questions
- Personal profile distribution after the page post has early engagement
